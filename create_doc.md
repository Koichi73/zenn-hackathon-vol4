# ハッカソンアイデア：動画データから手順書の自動作成

## 背景

手順書作成は、手間のかかる作業の一つである。ITリテラシーが低い人を考慮すると、全てのステップを記載したり、画面キャプチャを追加したりなど、さらに作成コストは増える。

新しいサービスの会員登録を例とすると、以下のような作業が発生する：

- 全ての画面遷移をキャプチャ
- キャプチャ内の個人情報はマスク(画像編集)
- 画像内のわかりづらいボタンは、赤枠や赤文字などで補足(画像編集)
- 全てのステップで、テキストによる手順の説明
- エラーが起きた場合のトラブルシューティングの記載
- 分岐がある場合の解説
- 最終的に手順書通りに機能するか再確認


さらに、AWSなどアップデートが速い分野では、画面の変更が激しく、せっかく作った手順書がちょっと違うということもよく発生する。


## 競合調査

**第1・2世代：DOMイベントベース（Scribe, Tango）**
Webブラウザの裏側にあるHTMLコード（DOM）を解析し、操作ログとスクリーンショットを記録する手法です。
*   **特徴:** テキスト生成が正確で、編集も容易です。
*   **課題:** Figma等のCanvasアプリやリモートデスクトップ環境ではDOMが取得できず機能しません。また、マニュアルの更新はユーザーが手動で検知・作業する必要があり、プライバシー保護（マスキング）は正規表現や手動指定に依存しているため、漏洩リスクや手間の課題が残っています。

**第3世代：ビデオ解析・CVベース（Guidde, Trupeer）**
画面のピクセル情報（動画）をAIとOCRで解析する手法です。
*   **特徴:** アプリケーションの種類を問わず、画面に映るものであれば何でも手順化可能です。
*   **現状:** 出力は「動画マニュアル」が主体であり、静的なドキュメント生成の品質はDOMベースに劣る傾向があります。



## アイデアの本質

競合がまだ進出少ない「動画inputかつテキストoutput」の領域を狙う。
従来のAIではできなかったUXをGemini 3で実現する。


## 機能(必須)

- 動画の手順をテキストに変換する
- 動画から手順ごとの画像を切り抜く
- 画像内の個人情報をマスクする
- 画像内のボタンを強調する
- テキストを画像を合わせて手順書を作成する
- 動画データは学習されない
- 出力はマークダウン形式とpdf形式
- 手順書のテキストを手動で変更できる

### 機能(任意)

- 手順書の画像を手動で変更できる
    - 切り抜き画像の変更
        - アプリ内でフレーム(1sごと)を保持して、簡単に差し替えができる
    - マスク箇所の変更
    - 強調箇所の変更
    - テキストの変更
    - 上記を満たすには、画像生成ではなく、画像編集をコードベースで行う必要がある

## 技術スタック
- 生成AI: Gemini 3
    - 動画などのマルチモーダルに強いため
- 画像生成: nano banana pro
    - 質の高い画像生成が可能。また、マスクした際にその他の部分が崩れない
- Agent: ADK
- バックエンド: Fast API
- フロントエンド: Next.js
